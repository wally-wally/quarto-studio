# Quarto Code Cell Options Reference

Source: Quarto official cell option reference pages — [cells-knitr](https://quarto.org/docs/reference/cells/cells-knitr.html), [cells-jupyter](https://quarto.org/docs/reference/cells/cells-jupyter.html), [cells-ojs](https://quarto.org/docs/reference/cells/cells-ojs.html). These are the `#|` options placed inside code chunks, grouped by category and engine.

## Knitr

Options available when the `knitr` execution engine is used (R code cells).

### Attributes

| Option | Description |
|--------|-------------|
| `label` | Unique label for code cell. Used when other code needs to refer to the cell (e.g. for cross references). |
| `classes` | Classes to apply to cell container. |
| `renderings` | Array of rendering names, e.g. `[light, dark]`. |

### Code Output

| Option | Description |
|--------|-------------|
| `eval` | Evaluate code cells (if `false` just echos the code into output). Accepts `true` (default), `false`, or a list of positive/negative expression numbers to selectively evaluate. |
| `echo` | Include cell source code in rendered output. Accepts `true`, `false`, `fenced`, or a list of line numbers. |
| `code-fold` | Collapse code into an HTML `<details>` tag so the user can display it on-demand. `true`, `false` (default), or `show` (open by default). |
| `code-summary` | Summary text to use for code blocks collapsed using `code-fold`. |
| `code-overflow` | Choose how to handle code overflow when lines are too wide for their container: `scroll` or `wrap`. |
| `code-line-numbers` | Include line numbers in code block output (`true` or `false`). |
| `lst-label` | Unique label for code listing (used in cross references). |
| `lst-cap` | Caption for code listing. |
| `tidy` | Whether to reformat R code. |
| `tidy-opts` | List of options to pass to `tidy` handler. |
| `collapse` | Collapse all source and output blocks from one code chunk into a single block (by default in pandoc markdown). |
| `prompt` | Whether to add the prompt characters in R code. |
| `class-source` | Class name(s) for source code blocks. |
| `attr-source` | Attribute(s) for source code blocks. |

### Cell Output

| Option | Description |
|--------|-------------|
| `output` | Include the results of executing the code in the output. Accepts `true`, `false`, or `asis` (treat output as raw markdown with no enclosing block). |
| `warning` | Include warnings in rendered output. |
| `error` | Include errors in the output (note that this implies that errors executing code will not halt processing of the document). |
| `include` | Catch all for preventing any output (code or results) from being included in output. |
| `panel` | Panel type for cell output (`tabset`, `input`, `sidebar`, `fill`, `center`). |
| `output-location` | Location of output relative to the code that generated it. One of `default`, `fragment`, `slide`, `column`, `column-fragment`. |
| `message` | Include messages in rendered output. Accepts `true`, `false`, or `NA`. |
| `results` | How to display text results: `markup`, `asis`, `hold`, or `hide`/`false`. |
| `comment` | Prefix to be added before each line of text output. |
| `class-output` | Class name(s) for the text/console output. |
| `attr-output` | Attribute(s) for the text/console output. |
| `class-warning` | Class name(s) for warning output. |
| `attr-warning` | Attribute(s) for warning output. |
| `class-message` | Class name(s) for message output. |
| `attr-message` | Attribute(s) for message output. |
| `class-error` | Class name(s) for error output. |
| `attr-error` | Attribute(s) for error output. |

### Figures

| Option | Description |
|--------|-------------|
| `fig-width` | Default width for figures generated by Matplotlib or R graphics. |
| `fig-height` | Default height for figures generated by Matplotlib or R graphics. |
| `fig-cap` | Figure caption. |
| `fig-subcap` | Figure subcaptions. |
| `fig-link` | Hyperlink target for the figure. |
| `fig-align` | Figure horizontal alignment (`default`, `left`, `right`, or `center`). |
| `fig-alt` | Alternative text to be used in the `alt` attribute of HTML images. |
| `fig-env` | LaTeX environment for figure output. |
| `fig-pos` | LaTeX figure position arrangement to be used in `\begin{figure}[]`. |
| `fig-scap` | A short caption (only used in LaTeX output). |
| `fig-format` | Default output format for figures (`retina`, `png`, `jpeg`, `svg`, or `pdf`). |
| `fig-dpi` | Default DPI for figures generated by Matplotlib or R graphics. |
| `fig-asp` | The aspect ratio of the plot, i.e., the ratio of height/width. When `fig-asp` is specified, the height of a plot is calculated from `fig-width * fig-asp`. |
| `out-width` | Width of the plot in the output document, which can be different from its physical `fig-width`. |
| `out-height` | Height of the plot in the output document, which can be different from its physical `fig-height`. |
| `fig-keep` | How plots in chunks should be kept: `high` (default), `none`, `all`, `first`, `last`, or a numeric vector. |
| `fig-show` | How to show/arrange the plots: `asis`, `hold`, `animate`, or `hide`. |
| `out-extra` | Additional raw LaTeX or HTML options to be applied to figures. |
| `external` | Whether to externalize tikz graphics by compiling them to PDF (for tikz devices). |
| `sanitize` | Whether to sanitize tikz graphics (escape special LaTeX characters). |
| `interval` | Time interval (number of seconds) between animation frames. |
| `aniopts` | Extra options for animations; see the documentation of the LaTeX **animate** package. |
| `animation-hook` | Hook function to create animations in HTML output (default `ffmpeg`, alternative `gifski`). |

### Tables

| Option | Description |
|--------|-------------|
| `tbl-cap` | Table caption. |
| `tbl-subcap` | Table subcaptions. |
| `tbl-colwidths` | Apply explicit table column widths for markdown grid tables and pipe tables that are more than 72 characters wide. |
| `html-table-processing` | If `none`, do not process raw HTML table in cell output and leave it as-is. |

### Panel Layout

| Option | Description |
|--------|-------------|
| `layout` | 2d-array of widths where the first dimension specifies columns and the second rows. |
| `layout-ncol` | Layout output blocks into columns. |
| `layout-nrow` | Layout output blocks into rows. |
| `layout-align` | Horizontal alignment for layout content (`default`, `left`, `right`, or `center`). |
| `layout-valign` | Vertical alignment for layout content (`default`, `top`, `center`, or `bottom`). |

### Page Columns

| Option | Description |
|--------|-------------|
| `column` | Page column for output. |
| `fig-column` | Page column for figure output. |
| `tbl-column` | Page column for table output. |
| `cap-location` | Where to place figure and table captions (`top`, `bottom`, or `margin`). |
| `fig-cap-location` | Where to place figure captions (`top`, `bottom`, or `margin`). |
| `tbl-cap-location` | Where to place table captions (`top`, `bottom`, or `margin`). |

### Cache

| Option | Description |
|--------|-------------|
| `cache` | Whether to cache a code chunk. Cached chunks are skipped on subsequent renders unless modified. |
| `cache-vars` | Variable names to be saved in the cache database. By default all variables created in the chunk are saved. |
| `cache-globals` | Variable names that are not created from the current chunk (e.g. created from previous chunks). |
| `cache-lazy` | Whether to `lazyLoad()` or directly `load()` objects from the cache. |
| `cache-rebuild` | Force rebuild of cache for chunk. |
| `cache-comments` | Whether to prevent comment changes from invalidating the cache for a chunk. |
| `dependson` | Explicitly specify cache dependencies for this chunk (one or more chunk labels). |
| `autodep` | Detect cache dependencies automatically via usage of global variables. |

### Include

| Option | Description |
|--------|-------------|
| `child` | One or more paths of child documents to be knitted and input into the main document. |
| `file` | File containing code to execute for this chunk (as an alternative to providing the code inline). |
| `code` | String containing code to execute for this chunk (as an alternative to providing the code inline). |
| `purl` | Include chunk when extracting code with `knitr::purl()`. |
